## 1. MediaDetailView surface breakdown

### 1.1 Modes (`MediaDetailPresentationMode`)

Defined at `MediaDetailView.swift:13-16`. Two cases:

- `.previewCarousel` — card lives inside an entry/carousel phase of `PreviewOverlayHost`. Vertical scroll is force-disabled (`MediaDetailView.swift:1730`), `allowActionRowInteraction` false, `showExpandedChrome` false. Logo crossfade duration is longer (0.38s vs 0.26s, `MediaDetailView.swift:687`).
- `.expandedDetail` — used in both pushed-navigation flows and the post-expand state of the preview. Distinguishes via `isExpandedPreviewFlow` (defined `MediaDetailView.swift:161` — true when `onPreviewExitRequested != nil && presentationMode == .expandedDetail`). Pushed nav has nil callback.

Computed switches driven by mode (`MediaDetailView.swift:185-204`):
- `heroBackdropScale`: 1.14 in preview/expanded-preview, 1.08 otherwise.
- `heroOverlayHorizontalInset`: 118 / 128 / 140.
- `heroLogoMaxWidth`: 620 / 520.
- `heroLogoSlotHeight`: 138 / 120.
- `heroActionRowTopPadding`: 32 / 24.

Reveal gating: `effectiveMetadataVisible` (`MediaDetailView.swift:218-224`) in preview/expanded-preview also requires `detail != nil` so genres/cast/summary appear together rather than flickering in piecewise. In pushed-nav mode it short-circuits to true.

### 1.2 Sections in render order

Three layered renderings inside one ZStack in `contentLayout` (`MediaDetailView.swift:1626-1740`).

#### Background layers (z-bottom → top, all behind the scroll view)
1. `backdropLayer` (`MediaDetailView.swift:655-667`) — fixed background, NOT inside ScrollView. `HeroBackdropImage` with kenBurns + parallax offset, scale 1.14/1.08, clipped to geo size, only drawn when `showsBackdropLayer` is true.
2. `backdropScrollOverlay` (`MediaDetailView.swift:672-681`) — gated on `scrollProgress > 0.01`; `.regularMaterial` rectangle whose opacity == scrollProgress. **The gate is the perf workaround** documented in the comment at `MediaDetailView.swift:673-675`: `regularMaterial` runs a UIVisualEffectView blur every frame on tvOS; gating removes it from the view tree during entry/paging.
3. Left-side gradient (`MediaDetailView.swift:1657-1670`) — text-readability vignette, fades with `effectiveVignetteVisible` over 0.6s.
4. Bottom gradient (`MediaDetailView.swift:1673-1689`) — 55% height anchored bottom, fades with same flag.

#### Scrolling content (single ScrollView)
5. `heroMetadataOverlay` (`MediaDetailView.swift:715-885`) — sits at `frame(height: heroHeight)` with `.id("scrollTop")` for scroll-to-top, then below-fold content. Sections inside the overlay, top to bottom:
   - **Spacer** (pushes content to bottom).
   - **Logo slot** (`MediaDetailView.swift:725-765`): renders `CachedAsyncImage(url: effectiveHeroLogoURL)`. On `.empty` falls back to `heroTitleText` ONLY if `hasDisplayedHeroLogoImage` is false — once a real logo has been displayed, `.empty` paints `Color.clear` instead of reverting to a title. Shadow is suppressed in `isPreviewCarousel` to avoid offscreen compositing (`MediaDetailView.swift:736-744`).
   - **Genre / type / rating row** (`heroMetadataRow`, `MediaDetailView.swift:938-962`).
   - **Description block** (`MediaDetailView.swift:771-832`) — three branches by `currentItem.kind`:
     - `.episode`: header "SxxExx · Title" plus description, separated when `shouldBlurHeroSummary` is true (Infuse-style spoiler hide).
     - `.show / .season`: tagline (italic) + overview, with season-fallback to parent show tagline/summary (see `effectiveOverview`, `MediaDetailView.swift:1101-1108`).
     - default (movie): tagline OR summary with blur applied.
   - **Quality row** (`heroQualityRow`, `MediaDetailView.swift:987-1019`): year · duration · star rating · `QualityBadge` per badge from `detail?.mediaSources.first?.qualityBadges()`.
   - **Up Next caption** (`MediaDetailView.swift:838-843`): only when `upNextCaption != nil`.
   - All five above wrapped in a fixed-height VStack (`frame(height: 420)`, `MediaDetailView.swift:845`) so peek distance is constant regardless of description length.
   - **Action row + Starring** (`MediaDetailView.swift:852-880`): HStack of `actionButtons` left + "Starring" trailing text from `detail.cast.prefix(3)`. Stays at `.opacity(effectiveMetadataVisible ? 1 : 0)` and is the only piece of metadata NOT faded by scroll (so the focus engine can keep it reachable, comment at `MediaDetailView.swift:850-851`).
   - The whole text VStack uses `.opacity(1 - scrollProgress)` (`MediaDetailView.swift:847`) to fade out as the user scrolls.

6. `belowFoldPage(geoHeight:)` (`MediaDetailView.swift:1746-1772`). Whole thing has `.opacity(belowFoldLoaded ? 1 : 0)` with 0.35s easeOut, and inherits `.opacity(effectiveMetadataVisible ? 1 : 0)` from the parent (so it ALSO fades during preview paging). Order:
   - **Padding shim** `Color.clear.frame(height: geoHeight)` so the below-fold starts exactly one viewport down.
   - **Season section / episode section** (extracted): seasonSection only when kind is show/episode (`MediaDetailView.swift:1751-1752`); episodeSection only for `.season` (`MediaDetailView.swift:1753`).
   - Wrapped with `.padding(.top, belowFoldHeaderReserveHeight)` (= `158 * scrollProgress`, `MediaDetailView.swift:648-650`) — reserves space for the centered title logo to slide into as the user scrolls.
   - **`recommendedRow`** (`MediaDetailView.swift:2927-2941`) — `MediaItemAgnosticRow(title:"Related")` when `recommendedItems` non-empty.
   - **`collectionRow`** (`MediaDetailView.swift:2943-2957`) — same component with `collectionName` title; conditional on non-empty + name present.
   - **`castCrewRow`** (`MediaDetailView.swift:2959-2968`) — `CastCrewRow(cast:directors:)` when either is non-empty.
   - **`belowFoldTitleLogo`** (`MediaDetailView.swift:889-926`) — overlaid on the same ZStack at `.top`, opacity driven by `belowFoldTitleOpacity` (= `(offset - 30) / 90` clamped, `MediaDetailView.swift:1724`). Falls back to large title text when no logo URL.

#### `seasonSection` (`MediaDetailView.swift:1776-1804`)
- Loader `ProgressView("Loading seasons...")` when `isLoadingSeasons`.
- Else `SeasonPillBar` (`.opacity(scrollProgress)` so it only appears after scroll begins, `MediaDetailView.swift:1790`).
- Below it `unifiedEpisodeList` — horizontal `LazyHStack` of `EpisodeCard` across all seasons in one row. Season-boundary detection inserts wider leading padding (56 vs 24, `MediaDetailView.swift:1833-1834`).

#### `episodeSection` (`MediaDetailView.swift:1890-1945`) — for `.season` kind
- For single-season shows uses `SeasonPillBar` with one pill (`shouldUseSingleSeasonPillHeaderInSeasonDetail`, `MediaDetailView.swift:1806-1808`).
- Otherwise plain "Episodes" title (`MediaDetailView.swift:1903-1906`).
- Horizontal LazyHStack of `EpisodeCard` with `.remembersFocus(key: "detailEpisodes")` (`MediaDetailView.swift:1942`).

### 1.3 Action buttons (`MediaDetailView.swift:1176-1335`)

Wrapped in a `ScrollView(.horizontal, showsIndicators: false)` with `.scrollClipDisabled()` so focus-expanded pills don't clip (`MediaDetailView.swift:1185-1194`). Disabled (whole row) when `!allowActionRowInteraction`.

Two branches: `playableActionButtons` if `isPlayableFromProvider` (`MediaDetailView.swift:1155-1157`, true when provider isn't TMDB), else `watchlistActionButton`.

**Playable buttons, in order (`MediaDetailView.swift:1200-1335`):**

| Button | id | Conditional | Function |
|---|---|---|---|
| Play (show/season) | `"play"` | kind in `.show`/`.season` | `presentPlay(for: nextUpEpisode)`. Label = `showPlayButtonLabel` ("Continue SxxExx" if in progress, else "Play"). `.disabled(nextUpEpisode == nil)`. |
| Shuffle | `"shuffle"` | kind in `.show`/`.season` | `Task { await shufflePlay() }` → fetches all episodes, shuffles, sets `selectedEpisodeItem = shuffled[0]`, `playFromBeginning = true`. |
| Play (movie/episode) | `"play"` | other kinds | `presentPlay(for: currentItem)`. Label = "Resume" if `effectiveIsInProgress`, else "Play". |
| Watched toggle | `"watched"` | always | `Task { await toggleWatched() }`. Icon `checkmark.circle.fill` if watched, else `checkmark`. Circle-expands-to-pill on focus. |
| Info / summary | `"summary"` | `hasReadableSummary` (kind in movie/show/season/episode AND `effectiveOverview` non-empty) | Opens `SummarySheet` via `showSummarySheet = true`. |
| Trailer | `"trailer"` | `detail?.trailerURL != nil` | `Task { await loadAndPlayTrailer() }`. |
| Audio | `"audioTrack"` | `showsPreplayTrackPickers` (movie/episode only) AND `!preplayAudioTracks.isEmpty` | Opens `TrackSelectionSheet(.audio)`. |
| Subtitles | `"subtitleTrack"` | same as audio + `!preplaySubtitleTracks.isEmpty` | Opens `TrackSelectionSheet(.subtitles)`. |

Play-button is special: it renders an inline progress bar (`playButtonLabel`, `MediaDetailView.swift:1525-1564`). Bar width 80pt; fills with `displayedProgress`. Color flips to black-on-white when focused. `.fixedSize(horizontal: true, vertical: false)` prevents the parent HStack squeezing the text. Time string is "remaining" when in progress, else total duration. There's no separate "Resume" button — the same Play button changes its label.

Watchlist branch (`MediaDetailView.swift:1481-1500`): single pill, toggles via `PlexWatchlistService.shared`. Label "Add to Watchlist" / "Remove from Watchlist" with bookmark icon.

`presentPlay(for:launch:)` (`MediaDetailView.swift:1165-1174`): if `promptResumeOrRestart` AppStorage is on AND `item.isInProgress`, show `.confirmationDialog` with three options (Resume from X / Start from Beginning / Cancel, `MediaDetailView.swift:552-568`). Otherwise calls launch immediately.

### 1.4 Sheets / overlays presented from this view

- `TrackSelectionSheet` (audio) — `MediaDetailView.swift:504-516`.
- `TrackSelectionSheet` (subtitles) — `MediaDetailView.swift:517-532`. Selection writes go through `persistAudioTrackSelection` / `persistSubtitleTrackSelection` (`MediaDetailView.swift:1425-1461`), which serialize via `pendingPreplayWrite` chain.
- `SummarySheet` — `MediaDetailView.swift:533-538`.

Not a SwiftUI presentation. Two VC types based on `useApplePlayer` UserDefault: `NativePlayerViewController` or `PlayerContainerViewController`.

## 2. PreviewOverlayHost surface breakdown

### 2.1 Geometry per phase

In `PreviewOverlayHost.body` (`PreviewOverlayHost.swift:120-237`). For each visible index, `PreviewCarouselCard` is placed at `frame.midX, frame.midY` via `.position(...)`.

- `topInset = 52`, `cornerRadius = 28`, `centeredHorizontalInset = 88`, `sideCardGap = 14`, `carouselParallaxFactor = 0.70` (`PreviewOverlayHost.swift:88-92`).
- `centeredFrame` (`PreviewOverlayHost.swift:123-130`): `cardWidth = geoWidth - 176`, `cardHeight = geoHeight - 52 + 28` (overflows bottom by cornerRadius so bottom corners stay hidden), positioned at `(geoWidth - cardWidth)/2, 52`.
- `fullFrame` = full geo.
- `entryFrame` = sanitized source-tile frame from `sourceFrames[request.sourceTarget]` (or `capturedSourceFrame`), clamped into bounds (`PreviewOverlayHost.swift:663-671`).

Frame selection per phase (`PreviewOverlayHost.swift:598-629`):
- `.entryMorph` (current card only): `entryFrame` → the source tile frame. Side cards aren't visible during entryMorph (visibleIndices returns only `[selectedIndex]` filtered, `PreviewOverlayHost.swift:111-114`).
- `.carouselStable`: `carouselFrame` — derived from `carouselSlotPosition` so each visible index gets `centeredFrame.minX + slot * (width + gap)` (`PreviewOverlayHost.swift:631-641`). Slot is `index - selectedIndex` at rest, interpolating across `pagingFromIndex` during paging via `pagingProgress` (`PreviewOverlayHost.swift:643-652`).
- `.expandingHero / .expandedHero / .detailsStable`: current card → `fullFrame` (entire screen, cornerRadius=0). Side cards keep their carousel positions but get opacity=0 (`PreviewOverlayHost.swift:685-696`).
- `.exiting`: like carouselStable for current, but with `motionLocked = true`.

Corner radius per index (`PreviewOverlayHost.swift:673-683`): 0 for the selected card when in expandingHero+, otherwise 28.

Opacity per index (`PreviewOverlayHost.swift:685-696`): selected card always 1; non-selected cards 1 in carouselStable, 0 in every other phase.

zIndex per index (`PreviewOverlayHost.swift:698-711`): in carouselStable computed as `100 - 10*|slotDistance|` so the center card always overlays its neighbors symmetrically. In expanded phases: selected = 2, others = 1. Comment at `PreviewOverlayHost.swift:704-706` notes this fixes a directional asymmetry where side-card overscan would otherwise draw over the center card in one paging direction.

### 2.2 State machine transition graph

`PreviewStateMachine` defined `PreviewContext.swift:73-147`. States: `.entryMorph → .carouselStable → .expandingHero → .expandedHero → .detailsStable → .exiting`.

| From | To | Trigger | Site |
|---|---|---|---|
| `.entryMorph` | `.carouselStable` | `completeEntryMorph()` inside `withAnimation(previewEntryAnimation)` immediately after `await Task.yield()` in `startEntryAnimation` | `PreviewOverlayHost.swift:265-269` |
| `.carouselStable` | `.expandingHero` | `beginExpand()` from user input (`onTapGesture`, `.return` key, PlayPause, down on `onMoveCommand`) | `PreviewOverlayHost.swift:375-379`; user input lines 183-192 |
| `.expandingHero` | `.expandedHero` | `finishExpand()` after `60ms + 60ms = 120ms` await from begin (the `previewExpandAnimation` duration is 0.35s but state flips after 120ms — chrome is shown next) | `PreviewOverlayHost.swift:393-396` |
| `.expandedHero` | `.detailsStable` | `markDetailsStable()` from `MediaDetailView`'s `onScrollGeometryChange` when offset > 10 (calls `onDetailsBecameVisible`) | `MediaDetailView.swift:1725-1727`; `PreviewOverlayHost.swift:164-168` |
| `.expandingHero/.expandedHero/.detailsStable` | `.carouselStable` | `exitAction()` returning `.collapseToCarousel` — driven by Menu button → bridge → `handleExit` | `PreviewOverlayHost.swift:421-471`, `PreviewContext.swift:137-146` |
| `.carouselStable/.entryMorph/.exiting` | (dismisses overlay) | `exitAction()` returning `.dismissOverlay` | `PreviewOverlayHost.swift:425-432` |
| any | `.exiting` | `beginExit()` set just before `onDismiss(request.sourceTarget)` is called | `PreviewOverlayHost.swift:430` |
| `.carouselStable` (motion lock toggle, no phase change) | — | `beginPaging()` / `finishPaging()` on left/right input | `PreviewOverlayHost.swift:318, 329` |

`motionLocked` is independent of phase: true during entryMorph, expandingHero, exit and during active paging; false in carouselStable rest and expandedHero. It's read by `MediaDetailView` to suppress hero backdrop motion (the comment on `isExpanded` at `PreviewContext.swift:82-89` notes the user perspective treats expandingHero as already "expanded").

### 2.3 Two-phase reveal cascade and `previewAnimationSettled`

`previewAnimationSettled` is `PreviewOverlayHost.swift:85` State. Forwarded into `MediaDetailView` via `PreviewCarouselCard` → `PreviewHeroSurface`. Defers the heavy data cascade until the spring + staged fades have visually completed.

**Entry cascade** (`PreviewOverlayHost.swift:252-295`):
1. Reset every fade flag, `previewAnimationSettled = false`, bump `metadataGate.begin()`.
2. `Task.yield()` then `withAnimation(previewEntryAnimation)` (`spring(response: 0.45, dampingFraction: 0.88)`) wraps `stateMachine.completeEntryMorph()` — the card morphs from entryFrame to carouselFrame and corners ease.
3. Sleep **350ms**, set `motionLocked=false`, animate `vignetteVisible=true` with `easeOut(0.6)`.
4. Sleep **400ms**, animate `metadataVisible=true` with `easeOut(0.34)`.
5. Sleep **340ms** (matches the prior fade duration so the visual fade is "done"), `previewAnimationSettled = true`.

Total budget before cascade: 350+400+340 = **1.09s** after Task.yield, plus a leading spring (~0.45s response). Comment at `PreviewOverlayHost.swift:286-294` calls out the timing as intentional: text fade visually completing means any cascade invalidation lands after the user perceives the animation as done.

**Paging cascade** (`PreviewOverlayHost.swift:297-353`): same shape with different durations.
1. Resuspend (`previewAnimationSettled = false`), fade metadata + vignette out, bump gate.
2. `withAnimation(previewPagingAnimation)` (`timingCurve(0.4, 0.02, 0.18, 1.0, duration: 0.78)`) drives `selectedIndex` AND `pagingProgress` together — card horizontal motion + inner parallax driven from one progress.
3. Sleep **780ms** (= duration), call `finishPaging()`, clear paging state.
4. Sleep **140ms**, animate `vignetteVisible=true` with `easeOut(0.26)`.
5. Sleep **70ms**, animate `metadataVisible=true` with `easeOut(0.48)`.
6. Sleep **480ms**, `previewAnimationSettled = true`.

Rapid paging keeps the gate stale until the user stops (each `page(by:)` bumps the gate, so each prior Task's `metadataGate.isCurrent(token)` check returns false and exits before flipping the flag — comment `PreviewOverlayHost.swift:312-315`).

**Expand cascade** (`PreviewOverlayHost.swift:355-415`):
1. Reset chrome / scroll flags, bump gate.
2. If `vignetteVisible == false`, animate it in with `easeOut(0.3)`.
3. `withAnimation(previewExpandAnimation)` (`easeInOut(0.35)`) wraps `stateMachine.beginExpand()` AND `focusedArea = nil` — card morphs to fullFrame, carousel input layer detaches.
4. Sleep **160ms**, if `metadataVisible == false`, animate it in with `easeOut(0.22)`.
5. Sleep **60ms** more, `stateMachine.finishExpand()` (phase → expandedHero).
6. Sleep **60ms** more, animate `expandedChromeVisible = true` with `easeOut(0.18)`, set `verticalScrollEnabled = true`.
7. Sleep **60ms** more, `previewAnimationSettled = true`.

The ordering note at `PreviewOverlayHost.swift:390-392, 405-409` is explicit: finishExpand BEFORE chrome visible so `presentationMode` flips to `.expandedDetail` before the chrome onChange handler runs (`MediaDetailView` sets focus from there).

**Collapse cascade** (`PreviewOverlayHost.swift:433-470`):
1. `easeIn(0.18)` fade out `expandedChromeVisible`.
2. Sleep **140ms**, `withAnimation(previewExpandAnimation)` wraps state machine return to carouselStable.
3. Sleep **380ms** (≈ 0.35s + cushion), unlock motion, ensure vignette + metadata are visible (animate if not), restore `focusedArea = .carousel`.

### 2.4 In-place item swap during paging

`pagingFromIndex` (`PreviewOverlayHost.swift:78`) records `selectedIndex` at paging start. `pagingProgress: CGFloat` is driven from 0→1 inside the same `withAnimation` as `selectedIndex` (`PreviewOverlayHost.swift:321-324`). `carouselSlotPosition(for:)` (`PreviewOverlayHost.swift:643-652`) interpolates each visible card's slot from `startPos = index - fromIndex` to `endPos = index - toIndex` by `pagingProgress`. The result is fed into both:
- `carouselFrame.x` (card translation)
- `parallaxOffset` (inner image translation, scaled by `carouselParallaxFactor = 0.70`).

`visibleIndices` (`PreviewOverlayHost.swift:110-118`) shows `[selectedIndex-1, selectedIndex, selectedIndex+1]` in carouselStable+. During paging the new neighbor is already in the visible set because `selectedIndex` is the destination index inside `withAnimation`.

Each card's content rendering during paging:
- The **selected card** (and non-selected during carouselStable) uses `PreviewHeroSurface` (= MediaDetailView, `PreviewOverlayHost.swift:715-800`). `enableDetailDataLoading` is true only for `isCurrent`, so passive cards keep stale state but still render the hero (gating in `MediaDetailView.swift:1574-1597`).
- **Non-current cards in carousel** use `PreviewCarouselSideCard` (`PreviewOverlayHost.swift:871-905`): a lightweight `HeroBackdropImage` with `scaleEffect(1.14)` and a rounded border. Comment at `PreviewOverlayHost.swift:862-870` explains the deliberate downgrade — side cards previously had their own `HeroBackdropCoordinator`/`.task(id:)` which contributed overhead during the entry window.
- Selected `PreviewHeroSurface` is wrapped with `allowsHitTesting(usesExpandedSurface && isCardExpanded)` (`PreviewOverlayHost.swift:774, 798`) so taps only land when expanded.

### 2.5 Background dim, scrim, blur

- Outermost ZStack background: `Color.black.ignoresSafeArea()` (`PreviewOverlayHost.swift:139`). No animated dim — instant black behind cards.
- Cards' `PreviewCarouselCard` masks itself with `UnevenRoundedRectangle` (top corners rounded, bottom flat) (`PreviewOverlayHost.swift:787-795`).
- A separate `PreviewCarouselStageWindow` overlay (`PreviewOverlayHost.swift:802-818`) draws only the rounded-rect stroke border on the current carousel card.
- The vignette gradients are inside `MediaDetailView`, drawn AS PART OF the card surface (`MediaDetailView.swift:1657-1689`). They fade in via `effectiveVignetteVisible`.
- The blur material rectangle (`backdropScrollOverlay`, `MediaDetailView.swift:672-681`) is also inside the card; it only renders when `scrollProgress > 0.01`.
- During expanded the card mask is no-op (cornerRadius=0 in `cardCornerRadius(for:)`), but the mask shape itself still mounts (`UnevenRoundedRectangle(topLeading:0, ...)`); SwiftUI is essentially clipping to a rectangle.

Carousel-input transparent layer (`PreviewOverlayHost.swift:176-193`): a `Color.clear.focusable(true)` sits on top of the cards when `stateMachine.isCarouselInputEnabled` (only entryMorph + carouselStable). Eats moveCommand left/right/down, tap, return, playPause. Pulled out of the tree once expanded so MediaDetailView gets focus events directly. There's also a top-level `.onExitCommand`/menu bridge via `PreviewMenuBridge` (see Section 6).

### 2.6 Notable host-side state-only items

- `capturedSourceFrame`: copied once from `sourceFrames` on appear (`PreviewOverlayHost.swift:197`) and only updated if it was nil (`PreviewOverlayHost.swift:226-230`). Locks the morph origin against parent re-layout.
- `metadataGate: PreviewLoadGate` (`PreviewContext.swift:149-161`) — monotonic counter. Each new cascade calls `begin()` to bump. Pending Tasks check `isCurrent(token)` after each sleep, exit if stale. This is the core mechanism that makes the cascade interruptible.
- `items: [MediaItem]` is mutable State seeded from `request.items` in `init` (comment `PreviewOverlayHost.swift:66-69` calls out that this must be in init, not onAppear, because the first body render happens before onAppear). The prefetch loop enriches stubs in place (`PreviewOverlayHost.swift:473-567`).
- Prefetch loop (`PreviewOverlayHost.swift:473-567`): n±2 window. Fires once `previewAnimationSettled` flips true and on every `selectedIndex` change. For each neighbor: hits `MediaProvider.fullDetail` (Plex) or `MetadataSource.itemDetail` (TMDB), enriches `items[i]` if ref still matches (race protection), warms `HeroBackdropResolver` + TMDB logo cache, and for shows preloads 8 episode thumbnails into `ImageCacheManager`. There's a parallel `prefetchLogoForSelectedItem` that fires immediately on appear (no gate) because logos are cheap and missing logos cause visible swaps.

## 3. Critical performance workarounds

Every notable one I found, by file:line.

### MediaDetailView.swift

- **`backdropScrollOverlay` gated on `scrollProgress > 0.01`** (`MediaDetailView.swift:672-681`). Comment: `.regularMaterial` is `UIVisualEffectView` on tvOS, blur runs every frame; removing the view from the tree during entry/paging is the only safe option.
- **`shadow` suppressed on hero logo in `isPreviewCarousel`** (`MediaDetailView.swift:736-744`). Comment notes the bottom vignette already hits 95% black so the shadow is invisible, but it would still force an offscreen compositing pass per frame while `heroMetadataOverlay` opacity is animating.
- **Body modifier chain split into three segments** for Swift type-checker (`MediaDetailView.swift:351-368, 371-585, 1571-1619, 1626-1740, 1746-1772`). The author calls this "type-checker expression-complexity budget" relief — splitting into `contentLayout` → `contentWithDataTasks` → `contentWithInteractionHandlers`. Comments at `MediaDetailView.swift:352-353, 1568-1570, 1744-1745`.
- **`detailLoadTaskID` keyed only on rating key** (`MediaDetailView.swift:172-181`). Comment is explicit: NOT including `shouldLoadDetailData` or `previewAnimationSettled` because `.task(id:)` would cancel-and-restart mid-paging, "the dominant paging-jank source." Those gates are checked inside the body via early return; a separate `.onChange(of: canRunDetailCascade)` (`MediaDetailView.swift:1608-1618`) re-invokes the cascade when they flip true.
- **`loadDetailData()` gated twice** (`MediaDetailView.swift:2146-2247`): (1) `shouldLoadDetailData`, (2) `previewAnimationSettled`. Passive side cards still call `syncHeroBackdrop()` so they show the right logo, then early return.
- **Parallel `async let` fans** in `loadDetailData` (`MediaDetailView.swift:2208-2233`) — collection+recommendations or seasons+episodes+nextUp executed in parallel.
- **`prefetchBelowFoldThumbnails`** (`MediaDetailView.swift:2253-2286`) warms ~12 thumbnails before flipping `belowFoldLoaded = true`. The intent is that `CachedAsyncImage` hits the synchronous memory-cache path on the first frame of the below-fold reveal.
- **`belowFoldLoaded` flip is a single gate** instead of per-section loaders (`MediaDetailView.swift:2246`) — sections all appear together rather than popping in one at a time.
- **`onScrollGeometryChange` over `PreferenceKey`** (`MediaDetailView.swift:1715-1728`) — scrollProgress is computed continuously from offset (not from a separate animated state), so the reserve-height padding grows in lockstep without compound animation overshoot.
- **`hasDisplayedHeroLogoImage` latching** (`MediaDetailView.swift:746-749`): once a logo image renders successfully it stays "remembered." Subsequent `.empty` cases paint `Color.clear` instead of reverting to the title text — prevents logo→title flash during paging.
- **`retainedLogoURL`** (`MediaDetailView.swift:132, 399-410`): mirrors heroBackdrop.session.logoURL but only resets when the item ID changes. Survives transient nil values from the coordinator.
- **`animation` set to nil in preview carousel** for the `.animation(...)` on `currentItem.ref.itemID` (`MediaDetailView.swift:883`) — disabled animations on metadata key changes during carousel paging.
- **`scrollClipDisabled()` everywhere** scroll-clipping would interfere with focus-driven pill expansion (`MediaDetailView.swift:1194, 1867, 1940`).
- **`fixedSize(horizontal: true, vertical: false)` on play button** (`MediaDetailView.swift:1563`) — prevents sibling pill expansion squeezing the time text into "43 / m" wrap.

## 7. Things SwiftUI does that UIKit will do differently

### 7.1 Reveal cascade

The cascade is **not** "first hero, then 100ms later actions, then 200ms later cast." It's actually:

1. Spring morph carouselFrame (entry) or full screen (expand) — visible.
2. After 350ms, vignette gradients fade in (0.6s).
3. After +400ms, the entire metadata text block fades in (0.34s). This includes title/logo, description, year/duration/badges, AND below-fold sections — they're all gated by the same `effectiveMetadataVisible` opacity.
4. After +340ms, `previewAnimationSettled` flips → data cascade starts → `loadDetailData()` runs → `belowFoldLoaded` flips → below-fold ZStack fades in (0.35s).

So at user level the appearance is: card morphs in → vignette darkens → text appears → (some time later) below-fold appears once data is loaded. Cast/episodes/related don't have individually staggered fade timings; they share `belowFoldLoaded` and inherit the same `effectiveMetadataVisible` from above. The pretty Apple TV+ "stagger per row" is NOT actually implemented — it's a single bulk fade.

The SwiftUI implementation gets the "feel" of staged reveal by gating large blocks on opacity flags and using the `metadataGate` cancellation so rapid input doesn't trigger flickers. There are no actual per-row delays; UIKit could reproduce the current behavior with two CABasicAnimation chains and a Task-cancellation pattern. Adding actual per-row stagger would be net-new in UIKit.

### 7.2 Workarounds, fight-SwiftUI comments

Verbatim notable comments:
- `MediaDetailView.swift:172-178`: "Task ID keyed only on the rating key. We deliberately do *not* include `shouldLoadDetailData` or `previewAnimationSettled` here ... Including them in the task ID would cause `.task(id:)` to cancel and restart mid-paging when `isCurrent` flips, which was the dominant paging-jank source."
- `MediaDetailView.swift:352-353, 1568-1570, 1744-1745`: body split for "Swift type-checker expression-complexity budget."
- `MediaDetailView.swift:427-434`: "For shows the Play button is `.disabled(nextUpEpisode == nil)`. The initial focus-set on chrome reveal writes `focusedActionButton = "play"`, but SwiftUI won't actually place focus on a disabled button — so the value is set but no view has focus. Once the detail loads and `nextUpEpisode` becomes non-nil, the button enables; we have to re-assert the focus binding (setting to nil first forces a re-evaluation) to make focus actually land." — this is a real "fight SwiftUI" pattern.
- `MediaDetailView.swift:673-675`: "`.regularMaterial` is backed by `UIVisualEffectView` on tvOS, which runs a blur render pass every frame. Gating on scrollProgress > 0.01 removes the blur view from the tree entirely during the entry/paging animation." — material is a known perf cliff.
- `MediaDetailView.swift:733-744`: shadow is "redundant" and is suppressed in carousel mode to avoid "the per-frame offscreen compositing pass the shadow forces while `heroMetadataOverlay` opacity is animating."
- `MediaDetailView.swift:1542-1547`: "lineLimit(1)+fixedSize prevents the parent HStack squeezing an adjacent focused button from wrapping the time text."
- `MediaDetailView.swift:1559-1563`: "Make the play button hold its intrinsic width. Without this the row's HStack shrinks the Play button to give space to a sibling that just expanded into a pill on focus."
- `MediaDetailView.swift:2154-2160`: cascade gating note that the onChange handler "will re-invoke this method once `previewAnimationSettled` flips to true."
- `PreviewOverlayHost.swift:66-69`: "Seeded in init from `request.items` so the very first body render (which happens BEFORE onAppear fires) sees the populated array — otherwise `items[selectedIndex]` in the entry-morph ForEach would index an empty array and trap."
- `PreviewOverlayHost.swift:704-706`: "Keep the card nearest the visual center on top so side-card overscan never draws above the center card in one paging direction. This preserves a flat carousel plane while fixing directional asymmetry."
- `PreviewOverlayHost.swift:862-870`: removed per-card `HeroBackdropCoordinator` from side cards because "the coordinator adds nothing here because the only thing the side card shows is the downsampled backdrop for ~600ms before the real hero surface takes over."
- `PreviewContainerViewController.swift:88-97`: blocks system dismissals on the UIKit container so the Menu press intercept can be the only path to exit.
- `DetailCardCarousel.swift:153-155`: "Carousel input layer: only active when NOT expanded. This prevents .onMoveCommand from eating MediaDetailView's focus events."

### 7.3 SwiftUI gotchas the current design tolerates

- Focus on disabled buttons doesn't land (workaround above).
- Body modifier chains can blow the type-checker (3-way split).
- `UIVisualEffectView`-backed materials cost per-frame work that has to be gated behind opacity > epsilon.
- `.task(id:)` cancels when the id changes — using it directly with derived gates causes mid-animation restarts.
- The first body render fires before onAppear, so any data needed by the first render must be set in init.
- `withAnimation` on a state machine struct mutation works, but only if you also mutate State the body reads — the current code mutates `phase` AND visibility flags inside `withAnimation`.
- `.navigationDestination` is a no-op without a `NavigationStack` ancestor — the modifier is conditional via `NavigationDestinationsModifier` (`MediaDetailView.swift:3726-3748`) and the preview overlay routes sub-item navigation through `onSubItemNavigation` callback instead, because it's presented via UIKit `present(_:)` outside any SwiftUI nav stack.
- Imperative player presentation (`MediaDetailView.swift:2107-2114`) reaches up through `UIApplication.connectedScenes` to find the topmost VC and `present` — a hard departure from SwiftUI's declarative model, made necessary by the UIKit player container needing to intercept the Menu button itself.
